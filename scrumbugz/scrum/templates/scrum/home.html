Scrumbugs: Agile Project Management with Bugzilla

  • This site allows the visualization of the details of the projects managed in Bugzilla.

    • Immediate Goals Of Scrumbugs

    • Reduce Work In Progress
    • Balance Demand Against Throughput
    • Prioritize

    Definitions

    • Backlog: A curated list of bugs from bugzilla, things we should do
    • Ready List A curated list of bugs in scrumbugs, things we can do
    • Sprint A specific timeframe to complete tasks, for us 2 weeks to start

    How Scrumbugs Works : Generally

    • PMs & Devs maintain a backlog and a list of ready tasks that can be worked on. Collectively, in an iteration meeting, teams review the ready tasks across projects and assign work.

    How Scrumbugs Works : The Code

    • Documentation

    How Scrumbugs Works : The Rules

    • Project Managers (PMs) & Developers (DV) may add a story to the backlog for a project and promote it to the ready list at any time
    • PMs should only move a story to the current sprint during an iteration meeting
    • DVs can move stories from the ready list to the current sprint at any time
